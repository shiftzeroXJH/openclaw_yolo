# 🧠 YOLO 自动迭代训练系统设计文档 v1

## 🎯 项目目标

构建一个基于 OpenClaw 的 YOLO 自动训练系统，实现：

* 对话式任务创建
* 自动训练与记录
* 训练结果结构化分析
* 飞书通知
* 人工确认 + 超时自动迭代
* 可控的参数优化流程

---

# 🟦 Part A：OpenClaw（LLM / Agent 侧）

## 1. 职责边界

LLM 只负责：

### ✅ 做什么

1. **解析用户需求 → 结构化配置**
2. **调用 CLI（而不是直接训练）**
3. **阅读 summary.json 做分析**
4. **生成下一轮参数建议（受限）**
5. **生成飞书通知内容**
6. **解释错误（辅助）**

---

### ❌ 不做什么

* 不直接调用 YOLO CLI
* 不解析原始训练日志
* 不负责等待/定时
* 不直接修改数据库
* 不自由修改任意参数

---

## 2. OpenClaw 调用接口设计（对 LLM 暴露）

### 2.1 创建任务

```bash
openclaw-yolo create-task \
  --dataset-root <path> \
  --pretrained <model> \
  --save-root <path> \
  --goal <metric> \
  --auto-iterate <true|false> \
  --confirm-timeout <minutes>
```

---

### 2.2 检查数据集

```bash
openclaw-yolo inspect-dataset --dataset-root <path>
```

返回：

```json
{
  "yaml_candidates": [
    "/data/a/data.yaml",
    "/data/a/detect.yaml"
  ]
}
```

---

### 2.3 启动训练

```bash
openclaw-yolo run-trial --experiment-id exp_001
```

---

### 2.4 获取训练摘要

```bash
openclaw-yolo get-summary --trial-id trial_003
```

---

### 2.5 生成下一轮建议

```bash
openclaw-yolo propose-next --experiment-id exp_001
```

---

### 2.6 应用建议继续训练

```bash
openclaw-yolo continue --experiment-id exp_001
```

---

## 3. LLM 决策逻辑

### 输入给 LLM 的内容（每轮）

```json
{
  "experiment_config": {...},
  "recent_trials": [...最近2~3轮...],
  "latest_summary": {...},
  "search_space": {...参数白名单...},
  "goal": {...},
  "status": "WAITING_USER_CONFIRM"
}
```

---

### 输出（严格结构化）

```json
{
  "decision": "continue",
  "param_updates": {
    "imgsz": 768,
    "epochs": 150
  },
  "reason": "recall不足且未收敛"
}
```

---

## 4. 参数修改约束（非常关键）

LLM 只能从白名单中选：

```json
{
  "imgsz": [512, 640, 768],
  "batch": [8, 16, 32],
  "epochs": [50, 100, 150],
  "lr0": [0.01, 0.005, 0.001],
  "weight_decay": [0.0005, 0.001],
  "mosaic": [0.0, 0.5, 1.0]
}
```

规则：

* 每轮最多改 1~3 个参数
* 不允许改未声明参数
* 不允许非法组合

---

## 5. 飞书通知（由 LLM生成文案）

### 模板 1：训练完成

* 本轮指标
* 与上一轮变化
* 参数变化
* LLM 分析
* 下一步建议

---

### 模板 2：等待确认

用户可回复：

* `继续`
* `停止`
* `只改xxx`

---

### 模板 3：超时自动继续

说明：

* 已等待 60 分钟
* 自动应用策略继续训练

# 🟩 Part B：代码端设计（核心系统）

---

## 1. 总体架构（4层）

### 1️⃣ 对话输入层（由 OpenClaw 驱动）

→ 不在代码端实现

---

### 2️⃣ 执行层（核心）

负责：

* 参数校验
* 数据集扫描
* 调用 YOLO Python API

### 3️⃣ 状态机层

控制流程：

```text
INIT
↓
READY
↓
TRAINING
↓
ANALYZING
↓
WAITING_USER_CONFIRM
↓
AUTO_TUNE_PENDING
↓
RETRAINING
↓
COMPLETED / FAILED
```

---

### 4️⃣ 记录层（数据库）

## 2. 训练执行（Python API）

```python
from ultralytics import YOLO

model = YOLO(config.pretrained)

results = model.train(
    data=config.dataset_yaml,
    epochs=params.epochs,
    imgsz=params.imgsz,
    batch=params.batch,
    project=config.save_root,
    name=trial_id
)
```

## 4. 摘要生成规则（关键）

### 4.1 plateau 判断

```python
if recent_best - old_best < 0.002:
    plateau = True
```

---

### 4.2 overfitting 判断

```python
if train_loss ↓ and val_metric ↓:
    overfitting = True
```

---

### 4.3 loss 趋势

* 稳定下降 → stable_down
* 震荡 → oscillating
* 发散 → diverging

---

## 5. 自动迭代控制

### 5.1 停止条件

```json
{
  "max_trials": 8,
  "max_no_improve": 3,
  "min_delta": 0.003,
  "target_score": 0.65
}
```

---

### 5.2 超时继续机制

数据库字段：

```text
wait_until = now + 60min
status = WAITING_USER_CONFIRM
```

后台 worker：

```python
while True:
    exps = find_timeout_experiments()
    for exp in exps:
        auto_continue(exp)
    sleep(30)
```

## 6. 错误处理策略

### CUDA OOM

```text
batch 16 → 8 → 4
imgsz 640 → 512
```

---

### NaN

```text
lr0 ↓
关闭 mosaic / mixup
```

---

### 数据问题

```text
跳过坏图
提示用户
```
